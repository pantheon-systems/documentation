Terminus [4.3.2](https://github.com/pantheon-systems/terminus/releases/tag/4.3.2) is now available. This release adds GitLab support for VCS commands, a new `--custom-domains` flag for `drush:aliases`, and custom build path support.

## Key improvements in this release

- **GitLab support for VCS commands**: `vcs:connection:add` and `site:create` now support GitLab as a VCS provider, enabling customers using GitLab to manage repository connections via Terminus. ([#2873](https://github.com/pantheon-systems/terminus/pull/2873))
- **`--custom-domains` flag for `drush:aliases`**: A new flag allows including custom domains in generated Drush alias files. ([#2779](https://github.com/pantheon-systems/terminus/pull/2779))
- **Custom build path support**: You can now pass a custom build path when creating sites. ([#2851](https://github.com/pantheon-systems/terminus/pull/2851))

## How to upgrade to Terminus 4.3.2

If you use Homebrew (macOS-only) to manage your Terminus installation, you should upgrade using:

```shell{promptUser: user}
brew upgrade terminus
```

If you installed Terminus directly from the `.phar` file, you should upgrade using the `self:update` command:

```shell{promptUser: user}
terminus self:update
```

For more information about this release, visit the [GitHub release page](https://github.com/pantheon-systems/terminus/releases/tag/4.3.2).

If you have questions or concerns around Terminus, please use the [Terminus issue queue](https://github.com/pantheon-systems/terminus).
